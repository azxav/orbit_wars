# JAX PFSP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add PFSP self-play to `orbit_ppo_jax` without wrapping Python/Kaggle rollout agents or breaking JAX `jit`/`vmap` training.

**Architecture:** PFSP state lives in Python-side manifest/stat/bank modules, while the compiled PPO update receives fixed-shape JAX arrays for frozen policy parameters and match plans. Before PFSP is enabled, the trainer must stop assuming learner seat `0`; feature building, action decoding, value bootstrap, rewards, and ranks must all accept a dynamic learner seat.

**Tech Stack:** Python dataclasses and JSON manifests, JAX/Optax PPO, existing `orbit_jax_env` reset/step/simple heuristic/proxy policies, existing `orbit_ppo_jax` BC parameter trees and checkpoint format, pytest.

---

## Project Findings

- `orbit_ppo_jax/train.py` currently hardcodes learner seat `0` in `_learner_act`, `_value_for_state`, rollout action replacement, reward storage, and rank storage.
- `orbit_ppo_jax/features.py` and `orbit_ppo_jax/actions.py` currently call `int(player_id)` / `int(seat)` in JAX-facing helper paths, so dynamic per-env learner seats are not jittable yet.
- `train.py` rejects any opponent except `simple_heuristic_jax` and `jax_proxy`; CLI choices match that restriction.
- Existing checkpointing already stores fixed JAX parameter trees as `params.npz`, `config.json`, and `metrics.json`; PFSP should reuse this for promoted PPO snapshots.
- Existing tests in `tests/test_ppo_jax_readiness.py` already provide a `_tiny_bc_checkpoint()` fixture, tiny train coverage, source-cap coverage, checkpoint roundtrip coverage, and GAE tests. PFSP tests should extend that file or a new focused `tests/test_ppo_jax_pfsp.py` while reusing those patterns.
- The pasted plan is correct about avoiding Python agents inside JAX training. The main refinement is sequencing: dynamic-seat support must be implemented and tested before any league sampling or bank integration.

## File Structure

- Modify `orbit_ppo_jax/features.py`: remove Python integer conversion from feature helpers that are used inside JIT; keep `source_cap` static.
- Modify `orbit_ppo_jax/actions.py`: make action row construction accept scalar JAX seats.
- Modify `orbit_ppo_jax/train.py`: add config/CLI fields, split learner policy action helpers into seat-aware sampled and greedy variants, accept match plans/banks in update, store learner-seat rewards/ranks, and integrate PFSP lifecycle.
- Create `orbit_ppo_jax/pfsp.py`: manifest models, PFSP stats, sampling weights, match-plan construction, manifest roundtrip, stat updates, pruning decisions.
- Create `orbit_ppo_jax/pfsp_bank.py`: fixed-shape frozen BC parameter bank, checkpoint compatibility checks, slot refresh, tree stack/take helpers.
- Create `orbit_ppo_jax/pfsp_eval.py`: JAX-native evaluation matrix for current policy against anchors and frozen snapshots.
- Create `tests/test_ppo_jax_pfsp.py`: focused tests for dynamic seats, match plans, manifest/sampling/bank behavior, rotated reward selection, frozen opponent acceptance, and tiny PFSP training.
- Optionally modify `tests/test_simple_heuristic_jax.py`: add/adjust CLI parse expectations for `--opponent pfsp_jax` if not covered in the PFSP test file.

Rules:
- `fixed0`: all learner seats are 0.
- `rotate`: learner seats are `arange(envs) % players`.
- `random`: learner seats use the supplied Python/NumPy RNG.
- Learner seat gets `OPP_NONE` and slot `-1`.
- 2P has one learner and one sampled opponent.
- 4P default `one_pfsp_two_anchors` uses one PFSP/frozen or hard opponent, one anchor, and one diversity opponent. Do not fill all non-learner seats with the same entry.

## Refinements Versus Pasted Plan

- Do the dynamic-seat refactor first. PFSP without that would silently preserve P0 bias and produce incorrect reward/rank attribution.
- Keep `source_cap` static. Making it dynamic is unnecessary and would break fixed JAX shapes.
- Add a compatibility helper around BC config keys before loading promoted snapshots into the bank; shape mismatch failures from `jnp.stack` are too late and too opaque.
- Introduce a tiny pure helper for learner reward/rank extraction so the rotated-seat bug is directly tested without needing a large rollout fixture.
- Start PFSP training with the initial BC frozen slot only and no promotion in the first tiny smoke test. Promotion/pruning is a separate lifecycle task.
- Keep `eval_vs_heuristic.py` external. It uses Kaggle/Python agents and should not become part of the JAX PFSP training hot path.
